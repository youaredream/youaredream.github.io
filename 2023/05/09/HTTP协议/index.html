HTTP 协议

计算机与网络设备要相互通信，双方就必须基于相同的方法。比如，如何探测到通信目标、由哪一边先发起通信、使用哪种语言进行通 信、怎样结束通信等规则都需要事先确定。不同的硬件、操作系统之间的通信，所有的这一切都需要一种规则。而我们就把这种规则称为协议（protocol）。

超文本传输协议（英语：HyperText Transfer Protocol，缩写：HTTP）是一种用于传输超媒体文档（例如 HTML）的无状态性的以请求/响应方式运行的应用层协议。

TCP/IP 协议族

在了解 HTTP 之前，我们先来简单了解一下 TCP/IP 协议族。

TCP/IP 协议不仅仅指的是TCP 和IP两个协议，而是指一个由 FTP、SMTP、TCP、UDP、IP 等协议构成的协议簇， 只是因为在TCP/IP协议中 TCP 协议和 IP 协议最具代表性，所以被称为 TCP/IP 协议

TCP/IP协议详解 - 知乎 (zhihu.com)

简单的 HTTP 协议

在 HTTP 中，请求访问文本或图像等资源的一端称为客户端，而提供资源响应的一端称为服务端。

客户端向服务端发送请求，服务端给客户端发送响应

网络传输本质上是字符串（又称为报文）的传输，请求和响应也不例外。

发送请求实际上是发送请求报文；发送响应实际上是发送响应报文。

URI 和 URL

HTTP 通过 URI(URL) 来定义互联网上的某个资源

URL 与 URI的区别

URL主要用于定位网络上的资源（可以理解为浏览器地址栏输入的内容就是URL）

URI是用于定义项目的标识

URL是URI的子集，我们在 WEB 开发中一般用 URL 的叫法

常见的 HTTP 请求方法

GET:获取数据

GET 方法用来请求访问已被 URL 识别的资源。指定的资源经服务器端解析后返回响应内容。

GET 方法可以带参数，参数拼接在 URL 后

GET 方法一般没有请求体

栗子：

POST:发送数据

虽然用 GET 也能发送数据（通过参数），但是一般不用 GET 方法进行传输，而是用 POST。虽然 POST 和 GET 很相似，但是 GET 的目的是获取数据，而POST的目的是发送数据。

更多方法

1 GET 请求指定的页面信息，并返回实体主体。
2 HEAD 类似于 GET 请求，只不过返回的响应中没有具体的内容，用于获取报头
3 POST 向指定资源提交数据进行处理请求（例如提交表单或者上传文件）。数据被包含在请求体中。POST请求可能会导致新的资源的建立和或已有资源的修改。
4 PUT 从客户端向服务器传送的数据取代指定的文档的内容。
5 DELETE 请求服务器删除指定的页面。
6 CONNECT HTTP/1.1 协议中预留给能够将连接改为管道方式的代理服务器。
7 OPIONS 允许客户端查看服务器的性能。
8 TRACE 回显服务器收到的请求，主要用于测试或诊断。
9 PATCH 是对 PUT 方法的补充，用来对已知资源进行局部更新 。

GET方法和POST方法

对比 GET POST
本质区别 从服务器段取得数据也能传送数据,只是用来设计告诉服务器,你到底需要什么样的数据 传送数据到服务器端
传输方式 以URL的方式传参/可见 通过HTTP post机制，放置在HTML HEADER内一起 传送到ACTION属性所指的URL地址/不可见
后退刷新时 后退刷新时无任何影响 后退刷新时会被重新提交
数据大小限制 受限制 不受限制
数据类型限制 只允许 ASCII 字符 不受限制，同样允许二进制数据（文件）
安全性 不安全 安全（相对）
请求体 一般无请求体 有
  1. GET提交的数据放在URL中，POST则不会。这是最显而易见的差别。这点意味着GET更不安全（POST也不安全，因为HTTP是明文传输抓包就能获取数据内容，要想安全还得加密）
  2. GET回退浏览器无害，POST会再次提交请求（GET方法回退后浏览器再缓存中拿结果，POST每次都会创建新资源）
  3. GET提交的数据大小有限制（是因为浏览器对URL的长度有限制，GET本身没有限制），POST没有
  4. GET可以被保存为书签，POST不可以。这一点也能感受到。
  5. GET能被缓存，POST不能
  6. GET只允许ASCII字符，POST没有限制
  7. GET会保存在浏览器历史记录中，POST不会。这点也能感受到。

HTTP的无状态性

HTTP 是一种不保存状态，即无状态（stateless）协议。HTTP 协议自身不对请求和响应之间的通信状态进行保存。也就是说在 HTTP 这个级别，协议对于发送过的请求或响应都不做持久化处理。

使用 HTTP 协议，每当有新的请求发送时，就会有对应的新响应产生。协议本身并不保留之前一切的请求或响应报文的信息。这是为了更快地处理大量事务，确保协议的可伸缩性，而特意把 HTTP 协议设计成如此简单的。

因此，如果需要保存登录状态等信息，光靠 HTTP 是不行的，可以使用 Cookie、Session、Token 等方法解决
